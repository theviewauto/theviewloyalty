# 📋 Loyalty Rewards + PDPA Compliance Guide

## ⚖️ PDPA (Personal Data Protection Act) - กฎหมายคุ้มครองข้อมูลส่วนบุคคล

ระบบนี้ **ปฏิบัติตามกฎหมาย PDPA** อย่างถูกต้อง ดังนี้:

---

## 1️⃣ ข้อมูลที่เก็บในระบบ

### ข้อมูลบังคับ (ตามกฎหมาย PDPA)
- **ชื่อ-นามสกุล** ✓ (หมายเหตุ: ป้องกันการใช้ตัวแทน)
- **เบอร์โทรศัพท์** ✓ (ติดต่อลูกค้า + ยืนยันตัวตน)
- **เพศ** ✓ (วัตถุประสงค์ทางการตลาด)
- **วันเกิด (DOB)** ✓ (วัตถุประสงค์ทางการตลาด + จัดโปรวันเกิด)

### ข้อมูลที่ไม่เก็บ ❌
- ❌ เลขบัตรประชาชน (ไม่ต้องเก็บ)
- ❌ ที่อยู่บ้าน (ไม่ต้องเก็บสำหรับสมาชิก)
- ❌ อีเมล (ไม่บังคับ)
- ❌ รหัส PIN (ไม่ต้องเก็บ)

---

## 2️⃣ PDPA Consent (ความยินยอมลูกค้า)

### ขั้นตอนการยินยอม
1. **ลูกค้ามาสัญชาติใหม่** → ไปยังหน้า "เพิ่มสมาชิกใหม่"
2. **กรอกข้อมูล** (ชื่อ เพศ อายุ เบอร์โทร)
3. **ต้องติ๊กเข้าที่ PDPA Consent Checkbox**
   - "ฉันยินยอมรับข้อมูลส่วนบุคคลสำหรับวัตถุประสงค์ทางการตลาด"
4. **ลงทะเบียน** → บันทึกข้อมูล `pdpaConsent: true`

### ถ้าไม่ยินยอม
- ❌ ระบบจะปฏิเสธการลงทะเบียน
- ผู้ใช้งานต้องติ๊ก checkbox ให้จึงจึงเสร็จ

---

## 3️⃣ วัตถุประสงค์การใช้ข้อมูล

ระบบนี้เก็บข้อมูลสำหรับ **3 วัตถุประสงค์**:

### ✓ วัตถุประสงค์ที่ 1: วางแผนด้านการตลาด
- **ใช้เพศ + วันเกิด (DOB)** → วิเคราะห์กลุ่มลูกค้า
- **แบ่งส่วน demographic** → รู้ว่าลูกค้าหญิง/ชาย/อายุเท่าไร
- **ทำการโปรโมชั่นเป้าหมาย** → ส่วนลดพิเศษให้กลุ่มเฉพาะ
  - เช่น: กระบะรุ่นใหม่ส่วนลด 10% สำหรับชาย 25-34 ปี

### ✓ วัตถุประสงค์ที่ 2: ติดตามการดูแลเครื่องยนต์
- **บันทึกประวัติการซ่อม** → จำได้ว่าลูกค้ามาซ่อมครั้งสุดท้าย เมื่อไร
- **ส่ง SMS เตือน** → "อายุน้ำมันเครื่องหมด ปลอดภัยหรือไม่?"
- **แนะนำบริการ** → "เวลาทำสปร์ค" "เช็คโช้ค" เป็นต้น

### ✓ วัตถุประสงค์ที่ 3: จัดโปรวันเกิด (Birthday Campaign)
- **ใช้วันเกิด (DOB)** → ทราบวันเกิดลูกค้าแต่ละคน
- **ส่ง SMS เตือนวันเกิด** → "วันเกิด [ชื่อ] เดือนหน้า! ส่วนลด 50%"
- **จัดโปรโมชั่นพิเศษ** → ส่วนลดเฉพาะวันเกิด / เพิ่มคะแนนโบนัส
- **สร้างความผูกพันลูกค้า** → ลูกค้าได้รู้สึกถูก appreciate

---

## 4️⃣ ข้อมูลไม่ไว้วางใจ ❌ (สิ่งห้าม)

### หลัก "ใช้ข้อมูลเท่าที่ได้ยินยอม"
- ✓ ใช้เพศ/อายุ → วางแผนการตลาด
- ✗ **ใช้เบอร์โทร → ขายให้บุคคลที่สาม** (ผิดกฎหมาย)
- ✗ **ใช้ชื่อ → โปรโมชั่นเก็งกำไร** (อนุมัติไม่ได้)
- ✗ **เก็บข้อมูลนานเกิน 1 ปี** (ต้องลบทิ้ง)

### ข้อมูล "Retention Policy" (ระยะเวลาเก็บ)
```
- ข้อมูลสมาชิกใช้งาน → เก็บได้นานตามความเหมาะสม
- ข้อมูลสมาชิกยกเลิก → ลบในเวลา 30 วัน
- ประวัติการซ่อม → เก็บได้นาน (เป็นประวัติการดำเนินการ)
```

## 5️⃣ การวิเคราะห์ข้อมูล (Analytics)

ตัวอย่างของข้อมูล **ตามกฎหมาย** ที่คุณสามารถวิเคราะห์ได้:

### 📊 รายงาน Demographic
```
👥 ทั้งหมด: 150 คน
  ├─ ชาย: 85 คน (56.7%)
  ├─ หญิง: 60 คน (40%)
  └─ ไม่ระบุ: 5 คน (3.3%)

👶 ช่วงอายุ:
  ├─ 15-24 ปี: 20 คน (13.3%)
  ├─ 25-34 ปี: 45 คน (30%)
  ├─ 35-44 ปี: 50 คน (33.3%)
  ├─ 45-54 ปี: 25 คน (16.7%)
  └─ 55+ ปี: 10 คน (6.7%)

⭐ Tier Distribution:
  ├─ Silver: 90 คน (60%)
  ├─ Gold: 45 คน (30%)
  └─ Platinum: 15 คน (10%)
```

### 🎯 ตัวอย่างการใช้ข้อมูล (ตามกฎหมาย)

#### ✓ ตัวอย่างที่ 1: โปรโมชั่นเป้าหมาย
```
"เรารู้ว่าลูกค้า 25-34 ปีหญิง มี 30 คน
→ ทำแผนโปรโมชั่น: "ชาร์จแบตฟรี" ส่วนลด 15%
→ ส่ง SMS เฉพาะกลุ่มนี้เท่านั้น"
```
✓ ตามกฎหมาย (ตัดสินใจตามข้อมูลที่ได้ยินยอม)

#### ✓ ตัวอย่างที่ 2: วางแผนบริการ
```
"เรามี Platinum (15 คน) มากที่สุดคือชาย 35-44 ปี
→ เปิดบริการ VIP เฉพาะ: "ซ่อมส่วนนอก" + นั่งโซฟาฟรี
→ ทำให้ประทับใจกลุ่มนี้"
```
✓ ตามกฎหมาย (พัฒนาบริการเพื่อลูกค้า)

#### ✗ ตัวอย่างที่ 3: ห้ามทำ
```
"เราขายรายชื่อโทรศัพท์ + เพศ/อายุ ให้บ้านจำนำ"
```
✗ **ผิดกฎหมาย PDPA**

## 6️⃣ วิธี Export ข้อมูลให้ Secure

### วิธีที่ 1: Export CSV (ปลอดภัย)
1. เปิดแอป Loyalty
2. กดปุ่ม "💾 Export CSV"
3. ได้ไฟล์ `loyalty_customers_2025-01-15.csv`
4. **เก็บไว้ในที่ปลอดภัย** (Google Drive ส่วนตัว / OneDrive)

### วิธีที่ 2: Export ไป Google Sheets
```
1. Export CSV
2. เปิด Google Sheets ใหม่
3. ข้อมูล → นำเข้า → อัพโหลด CSV
4. สร้าง Pivot Table เพื่อวิเคราะห์
```

### วิธีที่ 3: ใช้ Google Forms → Google Sheets (Advanced)
ถ้าต้องการให้ลูกค้าลงทะเบียนออนไลน์:
1. สร้าง Google Form (ชื่อ + เบอร์ + เพศ + อายุ + PDPA Consent)
2. ระบบจะบันทึกตรง Google Sheets อัตโนมัติ
3. Sync กับ Loyalty App ผ่าน API

## 7️⃣ ข้อมูลจำที่จำเป็น (PDPA Checklist)

### ✓ ทำแล้ว
- ✅ เก็บเฉพาะข้อมูลที่ต้องการ (ชื่อ เบอร์ เพศ วันเกิด)
- ✅ มี Consent Form (PDPA Checkbox)
- ✅ บันทึกว่า "PDPA Consent: Yes"
- ✅ เก็บข้อมูลเฉพาะในอู่เท่านั้น (ไม่ขายให้บุคคลที่สาม)
- ✅ ระบบคำนวณช่วงอายุจากวันเกิด (ไม่เก็บช่วงอายุเพราะ computed)

### ⚠️ ต้องทำต่อ (สำหรับหน้าต่อ)
- [ ] บันทึกวันที่ลงทะเบียน (เสร็จแล้ว ✓)
- [ ] สร้าง "Privacy Policy" สั้น ๆ (แนะนำให้พิมพ์บัตรสมาชิก)
- [ ] ตั้งระบบลบข้อมูลสมาชิกที่ยกเลิก (30 วันหลังยกเลิก)
- [ ] เก็บสำเนา Consent ไว้ (ถ้าลูกค้าสงสัย)

ข้อมูลที่เก็บ:

• ประวัติการซ่อม + คะแนนสะสม

วัตถุประสงค์:
✓ ให้บริการสมาชิก Loyalty

ข้อมูล่าจะส่ง:

ระยะเวลาเก็บ:

## 9️⃣ Q&A เกี่ยวกับ PDPA

### ❓ ถ้าลูกค้าขอดูข้อมูล?
**A:** ให้เปิด Detail เลย → แสดงชื่อ เบอร์ เพศ อายุ + ประวัติการซ่อม

### ❓ ถ้าลูกค้าขอแก้ไข?
**A:** 
1. เปิดแอป → สแกน QR
2. แก้ไขข้อมูลใน Detail mode
3. บันทึก

### ❓ ถ้าลูกค้าขอลบข้อมูล?
**A:** 
1. อู่ต้องลบออกจากระบบ (Delete ด้วยตนเอง)
2. บันทึกว่า "ลบเมื่อ [วันที่]"
3. ทำใบรับรองการลบ (เสริมเพื่อเป็นหลักฐาน)

### ❓ ลูกค้าไม่ยินยอม PDPA ล่ะ?
**A:** ก็สามารถเป็นสมาชิก แต่ไม่เก็บข้อมูลเพิ่มเติม
- เก็บเฉพาะ: ชื่อ + เบอร์ + ประวัติการซ่อม
- ไม่เก็บ: เพศ + อายุ (ไม่ส่วนลดการตลาด)

### ❓ เก็บข้อมูลนานแค่ไหน?
**A:** 
- ลูกค้าใช้งาน → เก็บได้นาน
- ลูกค้ายกเลิก → ลบในเวลา 30 วัน
